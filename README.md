# 🍎 水果智能识别服务系统 (Fruit-SmartID)

本项目是一个基于计算机视觉的**网页端水果智能识别服务系统**。用户只需上传水果图片或通过摄像头实时拍摄，系统即可自动识别水果种类，并化身“视觉计价器”与“健康生活助手”，一体化提供重量估算、保鲜建议、营养科普、产地应季查询及创意食谱推荐等全方位服务。

本系统具备实用性强、操作简单、响应式适配等特点，是一套非常完整的**计算机视觉全栈实训项目**。

---

## 🌟 核心功能特性

系统围绕核心 CV 算法，构建了 6 大上层业务功能：

1. **📷 水果种类识别（基础核心）**
   - 支持本地图片上传与实时摄像头拍摄。
   - 基于 YOLOv8 实现高置信度的多目标精准定位与识别（支持苹果、香蕉、橙子、梨、西瓜等常见水果）。
   - 前端同步输出可视化检测框、水果名称及置信度。

2. **⚖️ 视觉称重与估价（特色功能）**
   - 利用 OpenCV 进行轮廓检测与像素面积换算，自动估算水果重量。
   - 支持手动输入水果单价，联动计算单品及多件水果的总价，打造“视觉计价器”。

3. **❄️ 储存方式 & 保鲜建议**
   - 智能匹配最佳储存条件（常温/冷藏、适宜温度、保质期）。
   - 特设“禁忌提示”（例如：香蕉不可冷藏、芒果需避光），减少家庭水果浪费。

4. **🥗 食用禁忌 & 营养科普**
   - 详尽展示热量、糖分、维生素及膳食纤维等核心数据。
   - 提供空腹食用禁忌、过敏提示，并为减肥、糖尿病、老人及儿童等特殊人群提供科学食用建议。

5. **🗺️ 产地 & 应季查询**
   - 自动定位中国主产区，并结合当前系统时间智能打上 `【当季推荐】` 或 `【反季水果】` 标签。

6. **🍹 创意食谱 & 混合吃法**
   - 提供直接食用、榨汁、沙拉、甜品等六大分类食谱。
   - **支持多果组合推荐**：根据单次识别出的多种水果，智能组装并推荐混合果汁或精美拼盘食谱。

## 🛠️ 项目技术栈

项目采用前中后端分离架构，打通了从底层算法到上层视觉呈现的全栈流程：

| 模块 | 核心技术 | 说明 |
| --- | --- | --- |
| **前端 (Frontend)** | HTML5, CSS3, JavaScript, **Vue3** | 响应式布局（完美适配手机与电脑端），卡片式组件化展示 |
| **后端 (Backend)** | **Python**, **Flask / FastAPI** | 提供高性能的 RESTful API 接口，负责业务逻辑流转 |
| **视觉算法 (CV)** | **YOLOv8**, **OpenCV** | 目标检测（YOLOv8）、图像处理与轮廓像素面积换算（OpenCV） |
| **数据存储 (DB)** | **JSON** / **SQLite** | 用于存储结构化的水果营养成分、禁忌、产地及食谱扩展字段 |

## 🚀 快速开始

### 1. 克隆仓库

```bash
git clone https://github.com/yourusername/Fruit-SmartID.git
cd Fruit-SmartID
```

### 2. 后端环境配置与运行

确保本地已安装 Python 3.9+ 环境。

```bash
cd backend
# 创建并激活虚拟环境
python -m venv venv
source venv/bin/activate  # Windows 用户请使用: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动后端服务
python app.py
```

> **注意**：首次运行系统会自动检查并下载 YOLOv8 基础预训练模型，或请手动将训练好的 `best.pt` 放入 `weights/` 目录。

### 3. 前端环境配置与运行

```bash
cd ../frontend
# 安装依赖
npm install

# 启动本地开发服务器
npm run dev
```

打开浏览器访问控制台输出的地址（通常为 `http://localhost:5173`），即可直接调用本地摄像头或上传图片进行体验！

## 📐 核心算法简述

### 水果体积与重量估算逻辑

1. **图像预处理**：对经 YOLOv8 定位后的目标区域进行灰度化、高斯滤波及 Canny 边缘检测。
2. **轮廓提取**：利用 OpenCV 的 `findContours` 提取水果的闭合外轮廓。
3. **面积映射**：
   - 计算轮廓内部包含的像素点总面积 $A_{pixel}$。
   - *优化方案*：可在拍摄时引入标准参照物（如标准大小的手机或手掌），通过比例因子 $k$ 将像素面积转换为实际物理面积 $A_{real} = k \cdot A_{pixel}$。
4. **密度换算**：基于各类水果的几何拟合体积公式与平均密度数据库，实现从面积到重量（克/斤）的最终换算：

$$\text{Weight} = \text{Volume}(A_{real}) \times \rho_{fruit}$$
